W4JB Field Day Section Map

An interactive replay of the W4JB (Fayette County ARC) ARRL Field Day 2026 log, shaded by ARRL/RAC section. Proof-of-concept.

What you're looking at

Every contact in the club's 2026 Field Day log is plotted by the ARRL/RAC section reported in its exchange. Sections shade hotter as their QSO count grows, and briefly flash as new contacts come in during playback. The dataset is 1,180 QSOs spanning the full 24-hour contest window (2026-06-27 18:03Z → 2026-06-28 18:06Z), across five bands (10/15/20/40/80 m) and five modes (CW, FT8, MFSK, RTTY, SSB).

Using the map

  • Play / Pause replays the contest chronologically; the speed control sets how long the full 24 hours takes to play (about 30 s to 2 min).
  • Full contest jumps straight to the finished, all-contacts view.
  • The time slider scrubs to any moment; the map, counts, and panels all reflect that instant.
  • Select a section with a pointer, or focus it with Tab and press Enter or Space, to see its most recent contacts; hover provides a quick tally.
  • Band / mode chips filter what's shown.
  • Light / Dark switches the color theme. Dark is the first-visit default, and the selected theme is remembered on that device.
  • Alaska, Hawaii/Pacific, and Puerto Rico/Virgin Islands are drawn as inset boxes so the contiguous US stays large. DX and any section without map geometry appear in the "Off-map tally" panel rather than being dropped.

How the section map was built

ARRL does not publish an official GeoJSON of section boundaries, so the boundary file was built from public-domain sources: US Census Bureau 2023 Cartographic Boundary county polygons dissolved into sections using ARRL's published county-to-section list, plus Natural Earth province polygons for the Canadian RAC sections. Geometry is generalized to cartographic (national/continental) detail — appropriate for a section-level choropleth, not for street-level accuracy. The current file contains 81 valid worked section features; DX and the three invalid log values have no borrowed or placeholder geometry.

Known simplifications & data notes

  • Ontario is split by ARRL into four sections (GH, ONE, ONN, ONS), but no authoritative sub-provincial boundary data was available. The boundary file retains all four valid codes with the same province geometry; the interface deliberately draws one Ontario polygon and combines their heat, hover, flash, and detail totals. The detail panel still reports each subsection and each contact's actual code.
  • Pacific (PAC) is shown using Hawaii's shape only; Guam, American Samoa, and the Northern Mariana Islands are not represented.
  • Three logged section codes (NIL, WFL, SWF) are not current ARRL/RAC sections — each was flagged in the log itself as an invalid/garbled section copy (likely RTTY mis-decodes). They appear in the off-map tally as unrecognized rather than painted onto real geometry.
  • Single-operator log. The station/operator field is the club call on every record, so this dataset shows where W4JB reached, not per-operator performance. Per-operator identity would require per-op logging in the live setup.

Where this is headed

This is a Phase 1 proof-of-concept driven by a static export of this year's log. The same front end is designed to be fed by a live logging feed during next year's Field Day, so the map can update in real time as contacts are made.
